# 기본 코딩 원칙

## 작업 흐름
- 코드 작성 전 계획을 먼저 세우고, 확인 후 구현
- 구현 후 반드시 린트 실행하여 코드 품질 확인
- 테스트를 실행하여 의도대로 동작하는지 검증
- 변경된 코드가 기존 기능에 사이드이펙트를 일으키지 않는지 확인
- 위 검증을 모두 통과한 후에만 완료로 간주

## 코드 품질
- 하나의 함수/메서드는 하나의 역할만 수행
- 매직넘버 사용 금지, 상수로 정의
- 예외 처리는 구체적으로 (catch Exception 금지)
- 외부 API 호출부는 반드시 타임아웃과 재시도 로직 포함
- 민감 정보(API 키, 비밀번호)는 절대 하드코딩 금지

## 커뮤니케이션
- 한국어로 응답
- 변경 사항 설명 시 "왜" 그렇게 했는지 이유 포함
- 불확실한 부분은 가정하지 말고 먼저 질문

## 에러 대응
- 에러 발생 시 로그를 먼저 확인하고 원인 분석 후 수정
- 동일 에러 3회 반복 시 접근 방식을 바꿔서 시도
- 수정 후 관련 테스트 재실행으로 회귀 확인
